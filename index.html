What is PeopleSoft?

Oracle's Enterprise Platform
Built on Metadata

PeopleSoft is Oracle's ERP suite used by thousands of enterprises worldwide for HR, Finance, Supply Chain, and Education. What makes it unique is its metadata-driven architecture — every page, field, and workflow is defined as metadata in Application Designer, not hardcoded files.

HCM — HR & Payroll FSCM — Finance Campus Solutions PeopleTools IDE
🌐
Browser-Based via PIAThree-tier: Browser → WebLogic → Tuxedo App Server → Database. All business logic runs server-side.
🔧
PeopleTools — The Dev LayerApp Designer, PeopleCode, App Engine, Integration Broker, PS Query, BI Publisher — all built-in.
🗄️
Multi-Database SupportOracle DB, SQL Server, IBM DB2 — Meta-SQL makes your code database-independent.
☁️
On-Premise or Oracle CloudDeploy on your servers or OCI — DPK and PUM make setup and patching straightforward.

🧱 Module 4: Application Designer & Data Model

Effective Dating (EFFDT & EFFSEQ)

Instead of overwriting data when something changes, PeopleSoft inserts a new row with a new Effective Date — preserving the complete history of every change forever.

⚡ Key Takeaways
  • EFFDT = the date a row becomes valid
  • EFFSEQ handles multiple changes on the same date
  • PeopleSoft always shows MAX(EFFDT) ≤ today
  • Future-dated rows enter today but activate later
  • Historical rows are never deleted — ever
🏢 Real-World Context

In a 100,000-employee system, PS_JOB alone can have 3M+ rows. Every salary change, promotion, and transfer is a new row — not an update. This is what makes retroactive payroll possible.

⚠️ Common Mistakes
MISTAKE

Querying PS_JOB without effective dating WHERE clause — returns ALL rows for an employee (30+ rows), causing duplicate records in reports.

🧠 Mini Quiz Preview

What does PeopleSoft do when an employee gets promoted?

✅ Inserts a new row with the new Effective Date — the old row remains
