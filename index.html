// What is PeopleSoft?

An Enterprise Platform
Built on Metadata

PeopleSoft is Oracle's enterprise application suite powering HR, Finance, Supply Chain, and Campus Solutions for thousands of organizations worldwide. Founded in 1987 and acquired by Oracle in 2005, it is one of the most widely deployed ERP platforms in the world.

What makes PeopleSoft unique is its metadata-driven architecture — every form, field, workflow, and report is stored as metadata in system tables. Developers work in Application Designer to define objects that the runtime engine renders into working pages and reports. No raw HTML. No hardcoded SQL.

// Core Concepts You Will Master

The Building Blocks of PeopleSoft

01
🗂️
Component Processor
The runtime engine managing data flow — loading the buffer, firing PeopleCode events in sequence (RowInit → PostBuild → SaveEdit), writing back to DB.
02
📅
Effective Dating
Instead of overwriting records, new rows are inserted with EFFDT and EFFSEQ — enabling full historical reporting and retroactive payroll processing.
03
⚡
PeopleCode Events
PeopleCode attaches to a specific event — FieldDefault, RowInit, PostBuild, SaveEdit, SavePostChange. The event determines when your code fires.
04
🏗️
Metadata Architecture
Every record, page, component, and PeopleCode program is metadata stored in PeopleTools system tables — not files on disk.
05
🔒
Role-Based Security
User → Roles → Permission Lists → Row-Level Security. Permission Lists define which pages, processes, and data rows each user can access.
06
🔄
Integration Broker
Enterprise messaging platform for REST, SOAP, and async integrations. Service Operations, Nodes, Handlers, and Routing Rules form the full framework.
07
📱
Fluid UI
Responsive HTML5/CSS3 introduced in PT 8.53. Built on Oracle JET, works on mobile, tablet, and desktop with a modern grid-based layout system.
08
🛡️
Event Mapping
PT 8.55+ upgrade-safe customization. Attach custom App Class PeopleCode to delivered component events without touching Oracle's delivered objects.
